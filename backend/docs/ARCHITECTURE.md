## рџ§  Guidelines for Developers & AI Tools

This project follows a strict modular Django architecture.

### Rules:
- All business logic must live inside `src/apps/<app_name>/`
- Shared utilities must go into `src/common/`
- Project-level configuration must not be duplicated inside apps
- API logic must be inside `api/v1/` within each app
- Do NOT create global models or views outside apps

### File Placement Guide:
- Models в†’ `apps/<app>/models.py` (Must inherit from `TimeStampedUUIDModel` if generated by scaffolding)
- API Views в†’ `apps/<app>/api/v1/views.py`
- Serializers в†’ `apps/<app>/api/v1/serializers.py`
- Background tasks в†’ `apps/<app>/tasks.py` (Use `@shared_task` decorator)
- Signals в†’ `apps/<app>/signals.py`
- Forms в†’ `apps/<app>/forms.py`

### Environment vs Settings Boundary:
- `src/config/.env` controls infrastructure secrets (Ports, Passwords, Keys).
- `src/config/settings/` parses `.env` to determine logical variables (DEBUG, DATABASES).
- **Never hardcode secrets** into `settings.py` or `.py` files.

### Important:
Always prefer app-level modularization over global code.
